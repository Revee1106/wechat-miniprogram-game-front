# 微信小游戏主舞台卷轴重设计方案

日期：2026-04-07

项目：`wechat-miniprogram-game-front`

## 目标

重设计当前小游戏 Canvas 主舞台，使其：

- 保持原先小程序版本的纸墨、青绿、卷轴气质
- 明显优于当前“占位屏”式布局
- 更像一个真正的小游戏主舞台，而不是表单化信息页
- 兼容刘海屏与底部手势区，避免关键内容被遮挡

本次重设计只聚焦：

- 主舞台视觉结构
- 顶部题签
- 中央卷芯信息区
- 主按钮
- 底部功能签条
- 与抽屉 / 弹层的视觉衔接

不改变后端接口，不改变核心业务流程。

## 已确认方向

- 大方向：`山海卷轴`
- 风格要求：接近原小程序版本的气质，不做完全跳脱的全新风格
- 布局原则：单主场景 + 抽屉 / 弹层
- 适配要求：必须处理刘海、安全区和底部手势区

## 当前版本问题

当前 Canvas 主舞台虽然已经可运行，但主要问题是：

1. 信息块过于平面
   - 顶部、主体、底部像三块普通色块拼接
   - 缺少“卷轴”或“修仙主舞台”的视觉记忆点

2. 主次层级不清
   - 主按钮和功能入口都堆在底部
   - 没有明显的视觉中心

3. 结构过于像调试页
   - 更像一个开发阶段临时面板
   - 不像可长期演进的小游戏舞台

4. 刘海 / 安全区未成为明确布局规则
   - 虽然当前大概率还能显示，但并未把安全区作为设计约束写入布局模型

## 视觉概念

主舞台应被理解为“一幅正在展开的修行长卷”。

用户进入游戏时，不是看到一个 App 首页，而是看到：

- 顶部题签：点题
- 中央卷芯：承载当下命程
- 底部签条：进入副系统
- 异闻与结局：以浮层覆于卷面之上

记忆点应来自两个地方：

1. 居中的卷芯主卡
2. 底部四枚功能签条

## 新布局结构

## 1. 顶部题签区

顶部不再是纯色横条，而是“题签头”。

内容：

- 左上：`问道长生`
- 下方：一句短副标题
- 右上：预留轻量状态或系统按钮区

布局要求：

- 起始 y 必须基于 `safeTop`
- 顶部留白应为 `safeTop + 16~24px`
- 标题和副标题全部落在安全区下方

视觉要求：

- 顶部背景仍使用深青色，但不是纯矩形横条
- 可通过下沿弧线、题签边框或卷首阴影建立层次

## 2. 卷芯主区

中间区域做成一张“展开的卷芯”，替代当前普通白色卡片。

结构拆分：

- 上段：角色摘要
  - 境界
  - 寿元
  - 修为
  - 灵石
  - 轮次

- 中段：当前命程
  - 无事件时：行程提示
  - 有事件时：异闻预告
  - 已死亡时：终局提示

- 下段：主动作区域
  - `启程`
  - `推进时间`
  - 或其他当前唯一主操作

视觉要求：

- 卷芯边缘要有“纸张”感，而不是普通白底矩形
- 可通过双层阴影、边框、轻微泛黄、卷边处理实现
- 信息不能堆得太满，保留大片呼吸感

## 3. 主按钮

主按钮必须成为整屏唯一强焦点。

建议做法：

- 位于卷芯下部中央
- 形态接近印章 / 法印，而不是普通长按钮
- 尺寸明显大于底部签条入口

状态规则：

- 无 run 时：`启程`
- 正常进行中：`推进时间`
- 有事件时：不强化主按钮，由事件浮层优先接管
- 死亡时：由结算弹层接管，不再强调主按钮

## 4. 底部功能签条

`行囊 / 修行 / 洞府 / 炼丹` 不再像普通按钮，而是像卷轴下方垂挂的签条。

结构：

- 两列两行
- 每枚签条更窄更高
- 文字居中
- 与主按钮明显区分层级

交互：

- 点击后拉起对应抽屉
- 当前激活的签条在颜色或边框上有变化

布局要求：

- 必须基于 `safeBottom`
- 在底部手势区上方预留净空
- 不允许签条贴近屏幕底边

## 5. 抽屉与弹层的视觉衔接

当前既然主舞台是卷轴，则抽屉和弹层也要延续同一视觉语言。

要求：

- 抽屉头部有题签感
- 遮罩不要过黑，保持纸墨氛围
- 内容区可更密，但头部和边界仍需有卷页语义

区别：

- 事件弹层：更像“异闻覆页”
- 资源 / 修行 / 洞府 / 炼丹抽屉：更像“案卷抽匣”
- 结算弹层：更像“终卷”

## 安全区策略

## 顶部安全区

所有顶部元素都必须使用：

- `topStart = safeTop + 16`

并统一约束：

- 标题 baseline 不得进入刘海区
- 副标题必须低于标题至少一个正文行高
- 右上角轻操作区同样对齐安全区

## 底部安全区

所有底部按钮、签条、Toast 必须使用：

- `bottomSafe = safeBottom + 12~20`

约束：

- 功能签条整体上移，不能压在手势条区域
- Toast 不能与签条重叠
- 抽屉底部内容不能被底部手势区截断

## 适配策略

布局不走固定像素拼贴，而是遵循：

- 顶部题签高度区
- 卷芯主区
- 主按钮区
- 底部签条区

四段比例式布局，再叠加安全区偏移。

## 色彩与排版

## 色彩

保留当前基础色系，但强化层次：

- 深青：顶部、按钮、重点边框
- 纸黄：主背景
- 卷白：卷芯主卡
- 铜褐：签条、强调文字、异闻提示

避免：

- 现代 App 风格纯灰
- 霓虹或高饱和撞色
- 偏紫色方案

## 排版

标题：

- 粗重、稳，偏碑题或题签感

正文：

- 保持节制，行高略大

数字信息：

- 适合做成更规整的“命盘条目”

## 动效建议

本轮不追求复杂动画，但建议至少有：

- 首次进入时卷芯轻微浮现
- 抽屉上升时有稳定过渡
- 主按钮点击时有轻反馈

动效原则：

- 少而准
- 不用碎 micro-interaction 堆砌

## 验收标准

- 第一眼能看出“卷轴主舞台”的主题，而不是普通后台面板
- 视觉气质与原小程序版本保持连续性
- 顶部标题和底部功能区在刘海屏、手势屏上不被遮挡
- 主按钮成为整屏最明确的操作焦点
- 四个功能入口与主按钮形成明显层级差
- 事件、结算、抽屉与主舞台视觉语言统一

## 结论

本次主舞台重设计不是“换配色”，而是把当前小游戏首页重构成：

`题签头 + 卷芯主区 + 印章主按钮 + 底部签条入口`

在不改变核心业务结构的前提下，把当前页面从“可调试”提升到“可作为正式小游戏舞台继续扩展”的状态。
