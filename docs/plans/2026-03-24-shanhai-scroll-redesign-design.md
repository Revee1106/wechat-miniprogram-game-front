# 山海异志单线流程重设计方案

**日期：** 2026-03-24

**项目：** `wechat-miniprogram-game-front`

## 目标

把当前前端从“信息卡片堆叠式首页”调整为“单线主流程 + 辅助信息后置”的体验结构。新版必须优先满足以下原则：

- 非必要不滚动，主流程页面尽量一屏展示完成
- 每页只聚焦一类核心内容
- 每页只保留一个主按钮或一个主决策区
- 角色、资源、修炼、洞府等辅助信息不在主流程页常驻平铺
- 辅助信息通过查阅入口触发，优先使用底部弹层展示

## 范围

- 保留现有接口、数据字段和核心玩法流程
- 允许重排页面职责和跳转节奏
- 允许新增前端用的查阅弹层或弹窗组件
- 允许重写现有文案与状态提示
- 不新增后端接口
- 不扩展当前尚未落地的玩法功能

## 总体结构

新版主流程收敛为四个页面：

1. 封面页
2. 行程页
3. 事件页
4. 终卷页

辅助内容不再散落在首页，而是统一收进“查阅”入口：

- 命牌
- 行囊
- 修炼
- 洞府

查阅入口默认使用底部弹层，只有内容明显变长时才考虑独立跳页。

## 页面职责

### 1. 封面页 `pages/home/home`

定位为启动页，不承担信息总览职责。

页面只保留：

- 品牌标题“山海异志”
- 一句世界观引导文案
- 一个主按钮“启程”

页面不展示：

- 角色状态
- 资源条
- 修炼、洞府、炼制入口卡
- 结算信息

如果用户已经开局，封面页也不扩张成面板页，而是转为简洁入口，可通过主按钮进入行程页或继续当前行程。

### 2. 行程页 `pages/event/event` 或新增主流程页

定位为开局后的核心驻留页。

页面只展示：

- 当前已推进的月份或阶段
- 一句当前状态摘要
- 一个主按钮“推进时间”

页面不常驻展示角色、资源、洞府、修炼细节。

页面保留一个轻量“查阅”入口，点击后打开底部弹层。行程页的目标是让用户每次进入时都知道“我下一步只需要继续推演”。

### 3. 事件页 `pages/event/event` 或保留为独立页

定位为单次遭遇的决策页。

页面只展示：

- 事件标题
- 事件正文
- 当前可选项

风险、来源、状态等次级信息不再平铺成长面板，默认弱化为一行标签或“异闻注解”辅助入口。事件页的核心是让用户聚焦“发生了什么”和“我怎么选”。

### 4. 终卷页 `pages/summary/summary`

仅在角色死亡或寿终后出现。

页面只展示：

- 本局结果总结
- 永久成长
- 一个主按钮“转生再启”

终卷页不和存活时页面混用，不在主流程期间提供常驻入口。

## 查阅弹层

### 设计目标

把辅助信息从主流程页面抽离，但不让用户彻底失去查看渠道。

### 交互规则

- 由统一的“查阅”入口打开
- 优先使用底部弹层
- 弹层第一层只展示一个分区内容，不同时堆叠四组信息
- 弹层顶部允许切换：
  - 命牌
  - 行囊
  - 修炼
  - 洞府

### 信息约束

- 命牌：境界、修为、寿元、气运
- 行囊：灵石、药草、玄铁精华
- 修炼：当前是否可突破、突破条件提示
- 洞府：当前等级和简短加成说明

炼制由于当前仍是占位功能，不进入查阅弹层首层，后续如内容增加再评估。

## 视觉方向

继续沿用“山海异志”的旧纸、矿物青、赭石基调，但表现方式比上一版更克制。

视觉原则：

- 页面留白更多
- 标题区更像封面和章节页
- 主流程页不出现连续长卡片
- 单页只保留一个视觉中心
- 弹层承担信息密度，页面承担流程引导

## 文案策略

- 维持短句、克制、偏记述式表达
- 首页更像封面题签，不像功能面板
- 行程页文案强调“继续推演”
- 事件页文案强调“当前遭遇”
- 终卷页文案强调“此世已尽”

## 技术约束

- 保持原生微信小程序结构
- 尽量复用现有 store 和 API
- 如现有页面职责不适合新结构，可在前端层重命名页面定位，但不修改后端数据流
- 新增弹层时优先使用轻量组件，不引入额外依赖
- 继续修正乱码中文文案

## 验收标准

- 封面页一屏内完成展示，不需要滚动
- 行程页一屏内完成展示，不需要滚动
- 主流程页面不再常驻堆叠角色、资源、修炼、洞府信息
- 每个主流程页只有一个主按钮或一个主决策区
- 辅助信息可通过查阅弹层查看
- 终卷页仅在死亡后进入
- 当前中文乱码文案被修正
