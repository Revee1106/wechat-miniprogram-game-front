# 微信小游戏前端已完成功能说明

最后更新：2026-04-02

本文档仅记录 `wechat-miniprogram-game-front` 仓库中已经落地、且可从当前页面、状态管理与接口接入中确认的能力。

## 一、页面与基础流程

- 小程序已配置 6 个主页面：
  - 首页 `pages/home/home`
  - 行程/事件页 `pages/event/event`
  - 修行页 `pages/cultivation/cultivation`
  - 炼丹页 `pages/crafting/crafting`
  - 洞府页 `pages/dwelling/dwelling`
  - 结算页 `pages/summary/summary`
- 首页已支持创建单局运行，并在已有存档时直接续接流程。
- 当前局死亡时会自动跳转结算页；未死亡时从结算页返回主流程。
- 当前局在后端丢失时，前端已支持自动清理本地状态并回到首页。

## 二、行程/事件主界面

- 已接入当前事件展示与事件选项结算。
- 已接入时间推进操作。
- 已展示角色状态、修为、寿元、资源、洞府摘要等主信息。
- 已接入突破提示、突破门槛展示与突破按钮。
- 已提供事件历史记录，当前在前端缓存最近 2 条。
- 已提供洞府月结历史记录，当前在前端缓存最近 2 条。
- 已提供行囊资源卡片与统一底部动作面板。

## 三、资源动作与经济交互

- 已支持灵石转化修为。
- 已支持出售药草、灵植、灵矿、灵泉水等已接入资源。
- 资源动作已支持数量加减、半数、全部等快捷操作。
- 已在动作完成后弹出收益反馈 toast。

## 四、洞府页面

- 已接入洞府总览信息，包括洞府等级与当前灵石余额。
- 已展示设施卡片、设施状态、维护成本、产出与下一级花费。
- 已支持通过底部抽屉查看设施详情。
- 已支持建造与升级洞府设施。
- 已在推进时间前做“灵石不足将导致停摆”的风险提示确认。
- 已对常见建造/升级失败场景做前端友好报错翻译。

## 五、炼丹页面

- 已按“丹方 / 炉次 / 库存”三块面板组织移动端界面。
- 已过滤并展示当前可学习或可开炉的丹方。
- 已支持查看丹方材料详情。
- 已支持普通开炉与借灵泉开炉两种入口。
- 已展示炼丹库存，并支持直接服用丹药。
- 已接入炼丹相关资源读取，包括灵泉水库存。

## 六、修行与结算页面

- 修行页已展示角色、资源、结果摘要与突破门槛。
- 修行页已接入突破操作。
- 结算页已展示死亡后的结果摘要、角色状态与资源情况。
- 结算页已接入转生操作，并在转生后回到首页重新开始。

## 七、前端状态管理

- `utils/run-store.js` 已统一管理运行态、历史记录、错误信息与玩家轮回档案。
- 已为事件结算与洞府月结生成简明影响摘要，供页面展示。
- 已在创建新局、转生或后端 run 丢失时清理本地缓存，避免脏状态串局。

## 八、当前仍未覆盖为“完整体验”的部分

- 洞府月结历史目前仅保留前端最近 2 条缓存，尚未接入后端持久化历史列表。
- 资源价格仍是固定规则，尚未接入商店、订单或动态市场。
- 炼丹页已具备闭环，但尚未扩展到更复杂的批量排产、更多职业分支或更深的养成反馈。
- 战斗相关前端页面与交互尚未落地。
