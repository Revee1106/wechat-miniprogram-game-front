# 微信小游戏前端已完成功能说明

最后更新：2026-04-07

本文档仅记录 `wechat-miniprogram-game-front` 仓库中已经落地、且可从当前小游戏入口、Canvas 渲染层、状态管理与接口接入中确认的能力。

## 一、当前前端形态

- 前端主入口已经切换为微信小游戏模式，当前运行主路径是 `game.js`，不再以 `app.json + pages/**` 作为主入口。
- 新前端渲染层已经落到 `src/game/**`，采用原生 Canvas 2D、单主场景、抽屉 / 弹层结构。
- 现有 `utils/api.js` 与 `utils/run-store.js` 继续保留，作为小游戏前端的业务接入层。

## 二、主舞台与核心流程

- Canvas 主舞台已可显示角色核心摘要，包括境界、轮次、修为、灵石、寿元。
- 主舞台已接入“启程 / 推进时间”主按钮逻辑。
- 待处理事件时，前端会进入事件优先态，并通过事件弹层承载选项决策。
- 角色死亡后，主舞台会进入结算优先态，并显示转生弹层。

## 三、事件、资源与修行交互

- 已支持事件弹层展示标题、正文、选项、禁用原因，并接入事件结算动作。
- 已支持资源抽屉展示统一资源清单。
- 已支持在资源抽屉中执行出售资源与灵石转修为。
- 已支持修行抽屉展示当前境界、目标境界、突破条件与突破按钮。

## 四、洞府与炼丹交互

- 已支持洞府抽屉展示洞府等级、当前灵石、设施卡片、维护与收益摘要。
- 已支持在洞府抽屉中执行设施建造与升级。
- 已支持炼丹抽屉展示丹道熟练度、灵泉水数量、丹方列表、当前炉次与库存摘要。
- 已支持在炼丹抽屉中执行普通开炉、借灵泉开炉与服用丹药。

## 五、结算与转生

- 已支持死亡后的结算弹层展示结果摘要、当前境界、灵石、轮回次数与轮回点。
- 已支持在结算弹层中执行转生，并刷新为新一局状态。

## 六、前端架构状态

- `src/game/core` 已提供小游戏运行时、输入注册、基础布局与 UI 状态骨架。
- `src/game/view-models` 已覆盖主舞台、事件弹层、资源抽屉、修行抽屉、洞府抽屉、炼丹抽屉、结算弹层。
- `src/game/screens` 已覆盖以上各模块的 Canvas 绘制与主舞台接线。
- `src/game/adapters/run-store-adapter.js` 已统一承接开局、推进、事件结算、突破、洞府、炼丹、转生动作。

## 七、当前仍未完成的部分

- 旧小程序页面文件仍存在于仓库中，但它们已经不是主运行路径，后续还需要逐步退场或收缩为参考实现。
- Canvas 交互目前以最小可用为主，列表滚动、拖拽抽屉、动画与复杂文本排版仍需继续完善。
- 洞府与炼丹虽然已接入小游戏主舞台，但视觉层仍属于第一版迁移结果，后续还可继续压缩信息密度并细化反馈。
- 真机环境下的合法域名、HTTPS 与 TLS 联调仍需按小游戏发布要求单独处理。
## 2026-04-07 主舞台重设计补充

- 小游戏主舞台已从占位式块面重构为“题签头 + 卷芯主区 + 印章主按钮 + 底部签条入口”结构。
- 顶部标题与底部功能入口已按安全区重新布局，适配刘海屏与底部手势条。
- `行囊 / 修行 / 洞府 / 炼丹` 保持抽屉式交互，但视觉已与主舞台统一为纸墨卷轴风格。
- 事件弹层与结算弹层已同步使用新的纸色、铜色描边和深青强调色。
